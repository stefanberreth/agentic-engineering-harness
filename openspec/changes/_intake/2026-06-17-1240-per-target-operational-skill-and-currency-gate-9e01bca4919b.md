# Per-target operational skill artifact + a two-tier currency-gate ownership model (generic harness convention)

**Trigger:** A target needed a `/<slug>` operational skill so a fresh CLI / agent session can orient on the runtime OPERATIONAL-ACCESS surface (how to reach the running system, read-only query recipes, batch-routing decisions e.g. interactive-large-model vs token-budget CLI, deploy/cron mechanics, the env/credential surface by name, and the read-only-vs-mutation boundary) WITHOUT spelunking the codebase. The operator framed it as HORIZONTAL to functionality: whatever the project builds must stay reflected in this file, kept current, clean, non-redundant and thorough. They then asked the load-bearing governance question -- who keeps it current, and who keeps it from being silently abandoned.

**Insight:** This is a generalizable harness convention with two halves. (1) A per-target operational skill artifact: an orientation + index + recipes document that POINTS at authoritative docs (runbooks, deploy scripts, schedule definitions) rather than duplicating values -- thorough on the access surface, lean everywhere that lives elsewhere (the existing no-duplication / ground-truth-scan discipline applied to one file). (2) A currency-ownership model so it cannot silently rot, mapping onto EXISTING roles -- no new role: developer keeps it CURRENT (definition-of-done -- any change to the operational-access surface updates the skill in the same commit, or carries an explicit `skill-md: not-affected` attestation); reviewer keeps it CLEAN/coherent (cadence reconciliation -- completeness, consistency, non-redundancy -- bumping a `skill-md-last-reconciled` marker); orchestrator GUARDS abandonment (standing tracked obligation + the marker + refusal to sign off a phase boundary while the marker is stale -- the same non-discretionary mechanism as the mandatory reviewer cadence).

The key DESIGN lesson is to split currency enforcement by COST into two tiers: Tier 1 is a CHEAP DETERMINISTIC pre-push tripwire (a git-diff path check against a declared operational-surface allowlist; it blocks a push touching an allowlisted path without a skill touch or an attestation; NO LLM judgment). Tier 2 is the EXPENSIVE coherence judgment (does the skill completely/consistently/coherently reflect the system) run at the reviewer cadence, where judgment is affordable. The REJECTED design was a full coherence sweep on every push: a gate must be deterministic and cannot be an LLM judgment call (it rots to a rubber-stamp); whole-project coherence is a periodic job, not a per-event check; and most pushes do not touch the operational surface, so a per-push sweep is mostly waste.

**Suggested change:**
- Establish a harness convention (playbook + base-persona wiring + a hook template) for a per-target operational skill artifact plus the two-tier currency gate plus the developer/reviewer/orchestrator ownership split.
- Base-persona deltas (extension points so each target supplies its own allowlist/paths): developer base gains the skill-currency definition-of-done line; reviewer base gains the cadence reconciliation item + the `skill-md-last-reconciled` marker convention; orchestrator base gains the standing-obligation tracking + a phase-sign-off gate on a stale marker.
- A pre-push hook template implementing the deterministic operational-surface tripwire (declared allowlist + skill-touch/attestation check), riding the existing pre-push hook pattern. The allowlist EXCLUDES the agentic-engineering governance infra by construction so the gate never trips itself; the allowlist is per-target and operator-ratified (it governs the gate permanently); deliberately scope it to operational-access paths (deploy/schedule/topology/runtime-entry/env-wiring) and leave source + schema to the Tier-2 cadence to keep Tier 1 cheap and low-false-positive.
- Two caveats to document: (a) a local pre-push hook depends on per-clone git config (e.g. core.hooksPath) -- it is NOT a server-side push rule, so the DURABLE anti-abandonment backstop is Tier 2 + the orchestrator phase-gate, with Tier 1 as the cheap early-catch where configured; (b) the skill location must be git-trackable -- if the agent-config directory is gitignored wholesale, the tripwire is logically impossible and a precise negation is required (ignore-all-except-the-skills-subtree), verifying local-only markers stay ignored.
